INVESTIGATION METHODOLOGY

00

Phase 0: File Discovery (ตามหาไฟล์จากเบาะแส)

เริ่มต้นวิเคราะห์จากเบาะแสว่ามีคำว่า hackathon#1 ซ่อนอยู่ภายในระบบ แต่ไม่ทราบชื่อและตำแหน่งของไฟล์ จึงรันคำสั่ง Linux บน Server เพื่อค้นหาข้อความดังกล่าวแบบ Recursive จนระบุตำแหน่งไฟล์ Log ได้สำเร็จ

Hover to Analyze
grep -rn "hackathon#1" .
คำอธิบายคำสั่ง (Command Breakdown):
  • grep: คำสั่งหลักสำหรับค้นหาข้อความหรือแพทเทิร์นที่กำหนดในไฟล์
  • -r (Recursive): ค้นหาภายในไฟล์ของโฟลเดอร์ปัจจุบันรวมถึงโฟลเดอร์ย่อยทั้งหมด
  • -n (Line Number): แสดงเลขบรรทัดในไฟล์ต้นทางที่ตรวจพบข้อความตรงกัน
  • "hackathon#1": ข้อความเป้าหมาย (String Pattern) ที่ต้องการสแกนหา
  • .: ระบุโฟลเดอร์ปัจจุบัน (.) ให้เป็นจุดเริ่มต้นการค้นหา
01

Phase 1: Log Structure Profiling (สำรวจโครงสร้างไฟล์)

วิเคราะห์โครงสร้างคอลัมน์และลักษณะตัวกั้นเขตข้อมูล (Delimiter) ของไฟล์ Log ขนาดใหญ่ พบรูปแบบคั่นด้วยเครื่องหมาย Pipe | จัดแบ่งข้อมูลออกเป็น 7 ฟิลด์หลัก

Hover to Analyze
Timestamp | IP | Method | Path | Status | ResponseTime | Extra
โครงสร้างฟิลด์ข้อมูล (Field Breakdown):
  • Timestamp: วันและเวลาที่มีการส่งคำขอเข้ามา ใช้เรียงลำดับเวลาการโจมตี
  • IP: ที่อยู่ IP ของเครื่องต้นทาง ช่วยคัดแยกและวิเคราะห์พฤติกรรมผู้บุกรุก
  • Method: HTTP request method ที่ส่งเข้ามา (เช่น GET, POST, PUT, DELETE)
  • Path: เส้นทาง URL หรือ Endpoint ปลายทางที่ถูกเรียกใช้งาน
  • Status: รหัสตอบกลับ HTTP (เช่น 200 โหลดสำเร็จ, 500 ระบบล่ม, 504 ไทม์เอาต์)
  • ResponseTime: เวลาที่เซิร์ฟเวอร์ใช้ประมวลผล (หน่วยมิลลิวินาที) เพื่อวิเคราะห์อาการหน่วง
  • Extra: ข้อมูลเพิ่มเติมหรือ Payload ลายเซ็นที่ฝากมากับคำขอ
02

Phase 2: Threat Actor Isolation (หากลุ่ม IP คนร้าย)

นำเข้าข้อมูลลงใน Pandas DataFrame เพื่อจัดกลุ่มและนับจำนวนการเรียกขอ (Request Count) ของแต่ละ IP และคัดกรอง IP ที่มียอดเรียกขอสูงผิดปกติเกิน 10,000 ครั้งจนพบตัวตนแฮกเกอร์ 19 IPs

Hover to Analyze
ip_counts = df['IP'].value_counts()
hacker_ips = ip_counts[ip_counts > 10000]
ตรรกะโค้ด Python / Pandas:
  • df['IP']: เลือกเฉพาะคอลัมน์ IP Address จากตาราง DataFrame
  • .value_counts(): คำนวณความถี่และนับยอดคำขอของแต่ละ IP เรียงตามปริมาณจากมากไปน้อย
  • ip_counts > 10000: สร้างเงื่อนไขกรองเอาเฉพาะ IP ที่ส่งคำขอเข้ามามากกว่า 10,000 ครั้ง
  • ip_counts[...]: ดึงข้อมูล IP ตามเงื่อนไขเพื่อระบุรายชื่อ IP ของผู้บุกรุก

Phase 3: Attack Pattern Analysis (วิเคราะห์ Pattern การโจมตี)

ประมวลผลช่วงเวลาที่ระบบตอบกลับผิดปกติ คัดแยกข้อมูลตามช่วงเวลาชั่วโมงเพื่อหากรอบเวลาที่ระบบเกิดความขัดข้องสูงสุด (Status 500) และช่วงเวลาตอบกลับหน่วงสูงสุด (Response Time > 5000ms)

Hover to Analyze
crashes = df[df['Status'] == 500]
lags = df[df['ResponseTime'] > 5000]
ตรรกะโค้ด Python / Pandas:
  • df['Status'] == 500: สร้างเงื่อนไขดึงเฉพาะบันทึกที่มีสถานะการตอบกลับขัดข้องภายในระบบ
  • df['ResponseTime'] > 5000: สร้างเงื่อนไขเลือกเฉพาะบันทึกที่ตอบสนองช้ากว่า 5 วินาที
  • df[...]: กรองข้อมูลตามเงื่อนไขที่กำหนดเพื่อนำไปพล็อตไทม์ไลน์และเปรียบเทียบจุดพีก

Phase 4: Payload Tracing [BONUS] (หาเบาะแสจากข้อความที่คนร้ายทิ้งไว้)

สำรวจข้อความในคอลัมน์ Extra เพื่อหาข้อมูลผิดปกติ และระบุตัวตนหา IP ที่ฝาก Footprint ลายเซ็น hackathon#1 จนพบ IP หลักของคนร้าย 197.82.237.190 ที่มีการส่ง Payload ซ่อนไว้ใน URL

Hover to Analyze
weird_paths = df[df['Path'].str.contains('lnwza|secret|hack', case=False, na=False)]
ตรรกะโค้ด Python / Pandas:
  • df['Path']: เข้าถึงคอลัมน์เส้นทาง URL (Path) ที่มีการเรียกขอเข้ามา
  • .str.contains(...): ค้นหาคำสำคัญหรือคำเป้าหมายแบบ Regular Expression ภายในค่าคอลัมน์
  • 'lnwza|secret|hack': ลายเซ็นต้องสงสัยที่ค้นหา (ท่อนคำว่า lnwza หรือ secret หรือ hack)
  • case=False: กำหนดให้ไม่แยกแยะตัวอักษรพิมพ์เล็ก-พิมพ์ใหญ่ (Case Insensitive)
  • na=False: ให้แทนที่ค่าว่าง (NaN/None) ด้วยค่าเท็จเพื่อความปลอดภัยในการค้นหา
05

Phase 5: Reconstructing Signature [BONUS] (ถอดรหัสได้ข้อความลับ)

พัฒนาสคริปต์ดึงตัวอักษรพิมพ์ใหญ่ตัวสุดท้ายของแต่ละ URL เส้นทางปกติที่ตัด .html ทิ้ง นำมาเรียงต่อกันตามลำดับเวลา ยุบตัวหนังสือซ้ำ และแทนที่เครื่องหมายขีดล่างด้วยเว้นวรรคเพื่อเปิดเผยชื่อ GOEMON

Hover to Analyze
collapsed = []
for char in raw_message:
    if not collapsed or collapsed[-1] != char:
        collapsed.append(char)
final_message = "".join(collapsed).replace("_", " ")
ตรรกะกระบวนการถอดรหัสและลดรูปคำ:
  • collapsed = []: ประกาศอาเรย์ว่างเพื่อเก็บผลลัพธ์การตัดอักษรซ้ำ
  • collapsed[-1] != char: ตรวจสอบอักษรตัวปัจจุบันว่าต่างจากตัวก่อนหน้าที่เก็บหรือไม่
  • collapsed.append(char): เก็บตัวอักษรนี้ในอาเรย์เมื่อไม่เป็นตัวอักษรที่ซ้ำซ้อนกันติดต่อกัน
  • "".join(collapsed): นำผลลัพธ์ตัวอักษรทั้งหมดมาร้อยเรียงต่อกันเป็นคำเดี่ยว
  • .replace("_", " "): แทนที่สัญลักษณ์ขีดล่าง (_) ด้วยการเว้นวรรคเพื่อให้อ่านเข้าใจง่าย
